# Licensing

## Overview

LiturgyBridge contains different types of content and software components.

Each type may require different licensing considerations.

The project separates:

1. Software licensing
2. Liturgical content licensing
3. Translation licensing
4. Community content licensing
5. Media licensing

---

# 1. Software License

## Current decision

The LiturgyBridge software is licensed under:

Apache License 2.0

## Reason

The Apache License 2.0 provides:

- open collaboration
- permission for reuse
- protection for contributors
- suitability for community and professional use

The software can be used, modified, and distributed according to the license terms.

---

# 2. Liturgical Content

## Overview

Liturgical texts are different from software code.

The project must respect:

- copyright ownership
- publishing rights
- ecclesiastical approvals
- existing editions

## Principles

LiturgyBridge does not assume ownership of liturgical texts.

Every source must have:

- identified origin
- licensing information
- attribution requirements
- usage permissions

---

# 3. Original Liturgical Languages

Examples:

- Church Slavonic
- Greek
- Arabic
- Romanian
- Georgian

## Considerations

Traditional texts may have complex legal situations depending on:

- country
- edition
- publisher
- modern formatting
- annotations

The project should document the source of every text.

---

# 4. Translations

## Overview

Translations require separate consideration.

A translation may be:

- copyrighted
- published under permission
- created by contributors
- community-approved

## Principles

Every translation should include:

- translator information
- language
- source text
- license
- approval status

---

# 5. Community Contributions

## Overview

Communities may contribute:

- translations
- explanations
- documents
- photos
- recordings
- educational materials

## Principles

Contributors must have the right to share submitted content.

Communities remain responsible for content they publish.

---

# 6. Music and Hymn Resources

## Overview

Music materials require special attention.

Examples:

- hymn texts
- sheet music
- choir arrangements
- recordings

Possible rights holders:

- composers
- arrangers
- publishers
- recording creators

## Policy

Music resources require explicit licensing information before inclusion.

---

# 7. Images and Media

## Examples

- photographs
- videos
- audio recordings
- artwork

Each item should document:

- creator
- source
- license
- usage rights

---

# 8. Data Ownership

## Community Data

Community information belongs to the respective community.

Examples:

- member information
- internal documents
- group resources
- private media

## Principles

- communities control their own content
- private content must remain protected
- users control their personal data

---

# 9. Open Source and Open Content

## Long-term Goal

The project aims to encourage:

- open software development
- transparent documentation
- reusable resources

However, openness must respect:

- authors
- churches
- translators
- communities

---

# 10. Future Licensing Decisions

Before publishing large content collections, the project should define:

- accepted content licenses
- review process
- attribution standards
- contribution agreements

Possible future documents:

- CONTENT_POLICY.md
- TRANSLATION_GUIDELINES.md
- CONTRIBUTOR_AGREEMENT.md

---

# Summary

LiturgyBridge follows a simple principle:

Software should be open.

Content should be respected.

Tradition should be preserved.

Every contribution should be transparent and properly attributed.
